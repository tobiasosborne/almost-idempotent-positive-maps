# Plan v2: the argument as a typed module system — definitions DB · per-lemma af · enforced DAG · 200-LOC shards

## Context

Redesigned around four core principles you set (replacing my earlier lab-book-reorg-with-af-bolted-on):

1. **Definitions are PRIORITY 1.** A clean, **deduplicated** database of **provenanced, consensus-gated**
   definitions, each with a **local ground-truth copy**. If definitions drift, the project is sunk.
2. **Every lemma — no matter how small — gets its OWN af workspace.** Drive every result down to
   trivial steps. A workspace that grows a *huge* proof tree is a **failure signal**: the result has
   too many moving parts and is brittle → it must be factored into sub-lemmas.
3. **The entire ARGUMENT is managed as an enforced DAG** — exactly like code: **modules** (lemmas) with
   **contracts** (statements) between them. (You flagged: you don't know how to enforce this — that
   mechanism is the heart of this plan.)
4. **~200-LOC sharding everywhere**, with shards **easily findable** by agents. No monolith ever
   (the 2,444-line `agent-b-findings` and 91 KB compaction dump are the anti-pattern).

The mental model: **a typed module system for the proof.** Definitions = the *types/vocabulary*.
Each lemma = a *module* whose *contract* is its statement and whose *imports* are the lemmas/defs it
uses. Each af workspace = the module's *implementation*, kept tiny. A *linker* script enforces the
contracts (no drift, no cycles, no dangling imports, no oversized modules). Everything is sharded and
indexed so any agent finds the right module in one `grep`.

Confirmed feasible: `af get 1 -f json` (a workspace's root = its contract), `af externals -f json`
(its imports, with `Local: refs/...` provenance built in), `af status -f json` (node count = the
brittleness signal). A script can read all of this across every workspace.

## Architecture (4 layers)

```
definitions/            LAYER 0  the vocabulary — one shard per term (PRIORITY 1)
   <slug>.md            id, term, aliases, canonical statement, provenance(local), status, used-by

argument/               LAYER 1  the module graph — one shard per lemma (the DAG)
   lemmas/<id>.md       id, kind, contract(=statement), defs[], deps[], provenance, status, owner, workspace
   INDEX.md  DAG.md      GENERATED by the linker: id→path/statement/status; Mermaid dependency graph

proofs/<lemma-id>/      LAYER 2  the implementations — one tiny af workspace per lemma
   ledger/ externals/ meta.json   root conjecture == argument contract; imports == deps+defs

scripts/argument.py     LAYER 3  the LINKER + gate — validates contracts/DAG/drift/size; syncs beads
.beads/                          task DAG mirror of argument/ (one issue per lemma; bd deps = lemma deps)

refs/                   ground-truth: local copies (gitignored payload) + manifest/SOURCES.md (SHA256, tracked)
report/                 the paper — proof prose generated/checked FROM proofs/ exports
theory/  experiments/   working notes & numerics, sharded ≤200 lines, by subject
docs/ CLAUDE.md(==AGENTS.md) PRD.md HANDOFF.md GLOSSARY.md(→definitions/) archive/   lab-book + hygiene
lean/                   SECONDARY, later, fresh (af-tests = reference only)
```

## Layer 0 — Definitions DB  (principle 1; built FIRST)

`definitions/<slug>.md`, one term per file (the ultimate shard), YAML frontmatter + body:
```yaml
id: def-eps-jb-algebra
term: "ε-JB algebra"
aliases: ["epsilon-JB", "approximate JB algebra"]   # dedup keys
status: consensus            # draft → consensus (A+B sign-off recorded below)
provenance:                  # ground-truth must be a LOCAL copy, never an external abs path
  source: refs/hos/joa-m.md
  locus: "§3.1.4, line 2314"
  sha256: <16hex of refs/hos/joa-m.md>
  kind: cited                # cited | consensus | original
consensus: "A+B @ 2026-06-05"
used-by: [lem-bridge-jordan-identity, thm-bridge]    # generated back-refs
```
Body = the canonical statement (byte-verbatim from the local source for `cited`; the agreed text for
`consensus`/`original`).

- **Dedup + drift detector** (`scripts/check-defs.py`): two shards sharing a `term`/alias with
  differing statements → FAIL. One canonical definition per concept; report/af/theory all *reference*

- **Consensus gate:** a def is `consensus` only with A+B sign-off recorded; using a `draft` def in an
  af-validated lemma → WARN. (Consensus uses the existing verify-before-(proved) protocol.)
- **Local ground truth:** bring HOS, Kitaev, VLW, Effros-Størmer, Chu-Russo, Idel into `refs/` as
  local copies (payload gitignored, `refs/manifest/SOURCES.md` SHA256 tracked). Fixes today's broken

- **Seed:** inventory every definition currently scattered across `report/sections/02-05`,
  `agent-a/b-findings`, and `theory/` → collapse each to one canonical shard. (~15-25 defs:

## Layer 1 — Argument registry + the DAG/contract mechanism  (principle 3 — the novel part)

`argument/lemmas/<id>.md`, one **module** per file:
```yaml
id: lem-square-hole-almost-positive
kind: lemma                  # definition-use | lemma | proposition | theorem | corollary
contract: >                  # THE statement = the module interface (single source of truth)
  For r∈A, the square hole q_r = P(r²)−r² satisfies q_r ≥ −C·η·‖r‖²·𝟙.
defs:  [def-spectral-idempotent, def-eps-jb-algebra, def-kadison-inequality]
deps:  [lem-spectral-idempotent-properties]          # other lemma ids it imports
provenance: original         # or a refs/ source for a cited result
status: open                 # open → seeded → af-validated → consensus → report → lean
owner: B
workspace: proofs/lem-square-hole-almost-positive
```

**The contract/linker mechanism** (`scripts/argument.py` — the part you didn't know how to build):
- **Contract = one statement string**, living in the registry shard. It is used in two places, which
  the linker forces to agree: (a) the af workspace **root conjecture** (`af get 1 -f json`.statement),
  and (b) every *dependent's* import of it. So a lemma's statement cannot drift between where it's
  *proved* and where it's *used*.
- **Imports.** When lemma L depends on D, L's workspace does
  `af add-external --name "D.id" --source "<D's contract> | proof: proofs/D.id"`.
  The linker checks the embedded statement equals D's registry contract → **drift fails the build**.
  Defs used by L are `af def-add`'d and checked against `definitions/`.
- **The linker validates** (and `check-all.sh` fails the commit on any violation):
  1. **Acyclic** — `argument/` deps form a DAG (cycle ⇒ FAIL).
  2. **Contract match** — registry `contract` ≡ af root conjecture, for every seeded lemma.
  3. **Imports resolve** — every `dep` is a registry id; every af external naming a lemma matches its
     registry contract; every af def matches `definitions/`.
  4. **Status propagation** — L is `af-validated` only if its af root is `validated`+`clean` AND every
     internal-lemma dep is ≥`af-validated`. Computes the **ready frontier** (all deps done, itself open)
     and the **blocked set**.
  5. **Brittleness / 200-LOC analogue** — if a workspace's node-count or depth exceeds a threshold
     (default >12 nodes or depth >3), emit **REFACTOR: factor into sub-lemmas** (principle 2's failure
     signal, made mechanical).
  6. **Orphans** — registry lemma with no workspace, or workspace with no registry entry → FAIL.
- **Generated outputs:** `argument/INDEX.md` (id → path · contract · status · owner) and
  `argument/DAG.md` (Mermaid graph), regenerated by the linker; never hand-edited.
- **beads mirror:** the linker syncs one `bd` issue per lemma with `bd dep add` edges = registry deps,
  so `bd ready` *is* the proof frontier and `bd blocked` *is* the blocked set. Registry = math contracts
  (source of truth); beads = the queryable task/status DAG.

**Seed the registry BEFORE proving anything** — inventory every lemma/prop/theorem in `report/` +
`theory/` (~40-60 results), write a contract shard with `deps` for each. This *is* the architecture:
the whole argument becomes a visible, checkable DAG up front; brittle high-fan-in nodes are visible
immediately.

## Layer 2 — per-lemma af workspaces  (principle 2)

- `proofs/<lemma-id>/` = one `af init`'d workspace per registry lemma; root conjecture = the contract.
- Prove down to **trivial steps**. Mid-proof, if a sub-fact is non-trivial, DO NOT grow the tree:
  register it as a new lemma (`argument/lemmas/`), add it as an import (`af add-external`), and prove it
  in its **own** workspace. `af extract-lemma` promotes an over-grown node into its own module.
- Load-bearing algebraic identities: `af refine --crux` gated by `af claim-test --engine sympy`
  (reuses the project's existing numerics as evidence with SHA256 attachment).
- `proofs/README.md` documents this + the A/B role rule + `af reap` + ledger `.gitignore`
  (track `ledger/`, `externals/`, `meta.json`; ignore caches).

## Layer 3 — the gate (pre-commit "CI") + sharding/findability  (principle 4)

- **Pre-commit validation suite** `scripts/check-all.sh` (git `core.hooksPath=.githooks`; also the
  documented session-close checklist), each step exits non-zero on failure:
  1. `check-defs.py` — definitions dedup/drift + consensus-gate warnings.
  2. `argument.py --check` — the linker (acyclic · contract-match · imports · status · brittleness · orphans).
  3. `check-af-ledgers.sh` — `af replay --verify` in every `proofs/*` (content-hash integrity).
  4. `check-provenance.py` — every provenance row resolves to `refs/manifest` + SHA256 matches (warn if
     gitignored payload absent locally).
  5. `check-report-build.sh` — `latexmk -pdf -halt-on-error`, 0 undefined refs.
  6. (later) `lake build` 0-sorry/0-axiom once `lean/` is real.
- **Sharding & findability:** one def per file; one lemma per registry shard; theory notes ≤~200 lines;
  stable greppable ids (`def-<slug>`, `lem-<slug>`, `thm-<slug>`, `prop-<slug>`); the generated
  `argument/INDEX.md` + `definitions/INDEX.md` are the lookup tables; a `check-shard-size.sh` warns on
  any tracked `.md` > ~250 lines. Agents locate a module by `grep -rl "<id>"` or globbing the slug.
